# PulseLink Constitution

## Mission

PulseLink aims to reduce the time required to find blood donors during medical emergencies by instantly connecting patients with compatible and eligible donors.

## Core Principles

### Emergency First

Every feature must prioritize helping patients find blood donors as quickly as possible.

### Simplicity

Users should be able to register as donors or create blood requests within two minutes.

### Reliability

Blood requests, donor information, and donor responses must be stored accurately and securely.

### Privacy

Donor personal information should only be shared after the donor accepts a blood request.

### Fairness

The system should notify only eligible donors based on blood group, availability, and donation eligibility criteria.

### Scalability

The architecture should support future enhancements such as location-based matching, hospital integration, and real-time notifications.

## Technical Standards

* Frontend: React + Tailwind CSS
* Backend: FastAPI
* Database: MySQL
* REST API Architecture
* Mobile Responsive Design
* Secure API Communication
* GitLab Version Control
